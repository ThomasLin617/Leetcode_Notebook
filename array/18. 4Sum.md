# 18. 4Sum
## code
```python
class Solution:
    def fourSum(self, nums: List[int], target: int) -> List[List[int]]:
        nums.sort()
        ans = []
        for i in range(len(nums)-3):
            if i > 0 and nums[i] == nums[i-1]:
                continue
            for j in range(i+1, len(nums)-2):
                if j > i + 1 and nums[j] == nums[j-1]:
                    continue
                k = j + 1
                l = len(nums) - 1
                while k < l:
                    summ = nums[i] + nums[j] + nums[k] + nums[l]
                    if summ == target:
                        ans.append([nums[i], nums[j], nums[k], nums[l]])
                        k += 1
                        while k <l and nums[k] == nums[k-1]:
                            k+=1
                        l -= 1
                        while k<l and nums[l] == nums[l+1]:
                            l -= 1

                    elif summ < target:
                        k += 1
                        while k <l and nums[k] == nums[k-1]:
                            k+=1
                    elif summ > target:
                        l -= 1
                        while k<l and nums[l] == nums[l+1]:
                            l -= 1
        return ans
```
## idea
Compared to the 3Sum problem, solving 4Sum simply involves adding one more for-loopâ€”there's no fundamentally new concept introduced. However, it's easy to forget to move the two pointers after finding a valid combination, which is an important detail to watch out for. The time complexity is $O(n^3)$, which comes from two nested for-loops combined with an optimized two-pointer approach that runs in linear time $O(n)$.
